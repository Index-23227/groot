## 2. 소품 준비 체크리스트

### 약품 (대용품)

| 약품 | 대용품 제안 | 크기 | 요구사항 |
|------|-----------|------|---------|
| 파란 약병 | 파란 물병, 파란 마커 붙인 공병 | 높이 10~15cm, 지름 4~6cm | 그리퍼로 잡을 수 있는 크기 |
| 빨간 약병 | 빨간 물병 | 동일 | 파란색과 명확히 구분 |
| 하얀 약병 | 하얀 공병 | 동일 | 배경과 구분 가능해야 함 |
| 노란 약상자 | 노란 포스트잇 상자, 약상자 | 5×8×3cm | 병과 다른 형태 |

```
⚠️ 핵심 포인트:
- 색상이 카메라에서 명확히 구분되어야 함
- 그리퍼 너비에 맞는 물체 크기 (지름 4~6cm 권장)
- 너무 가볍지 않을 것 (바람에 날리면 안 됨)
- 표면이 미끄럽지 않을 것 (그리퍼 슬립 방지)
```

### 조제함 (대용품)

```
방법 1: 칸막이 상자
  ┌────────────────────┐
  │ ┌────┐┌────┐┌────┐ │
  │ │ 1  ││ 2  ││ 3  │ │   ← 칸당 약 10×10×8cm
  │ │    ││    ││    │ │      A4 종이 상자 + 칸막이
  │ └────┘└────┘└────┘ │
  └────────────────────┘
  각 칸에 번호 라벨 부착

방법 2: 트레이 3개
  [1] [2] [3]  ← 각각 작은 트레이/접시, 번호 표시
  간격 10~15cm

방법 3: 테이프 구역
  테이블에 마스킹 테이프로 3구역 표시 + 번호 적기
  (가장 간단, 소품 없어도 가능)
```

### 기타 필요 물품

```
□ 카메라 삼각대 (또는 모니터 암, 클램프)
□ USB 카메라 (또는 RealSense)
□ USB 연장 케이블 (2~3m, 카메라 위치 자유도)
□ 마스킹 테이프 (구역 표시, 카메라 고정)
□ 유성 매직 (번호 라벨)
□ A4 용지 + 매직 (1번, 2번, 3번 라벨 제작)
□ 마이크 (STT용, 노트북 내장 마이크도 가능)
□ (선택) 블루투스 스피커 (TTS 출력용)
```

## 3. 카메라 세팅

### 위치 결정

```
최우선: 로봇 뒤쪽 상단 45°
  → 그리퍼와 물체가 동시에 보임
  → 그리퍼에 의한 가림(occlusion) 최소화

차선: 로봇 측면 45°
  → 조제함 쪽에서 촬영
  → 약품 라벨이 보이는 각도

피해야 할 위치:
  ✗ 정면 (그리퍼가 물체를 가림)
  ✗ 바로 위 (depth 정보 없음)
  ✗ 너무 멀리 (물체가 작게 보임)
```

### 화각 확인

```bash
# 카메라 프리뷰로 화각 확인
python -c "
import cv2
cap = cv2.VideoCapture(0)
cap.set(cv2.CAP_PROP_FRAME_WIDTH, 640)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)
while True:
    ret, frame = cap.read()
    if not ret: break
    cv2.imshow('Camera Check', frame)
    if cv2.waitKey(1) == ord('q'): break
cap.release()
"
```

### 카메라 체크리스트

```
□ 약품 대기 구역 전체가 프레임 안에 보이는가?
□ 조제함 전체가 프레임 안에 보이는가?
□ 로봇 그리퍼가 프레임 안에 보이는가?
□ 각 약품의 색상이 구분되는가? (조명 확인!)
□ 조제함 번호 라벨이 읽히는가?
□ 카메라가 단단히 고정되어 있는가? (흔들림 없이)
□ 카메라 케이블이 로봇 동선에 걸리지 않는가?
```

### 조명 주의사항

```
⚠️ 대회장 조명 조건:
- 형광등 아래: 파란색과 하얀색이 비슷하게 보일 수 있음
  → 대비가 높은 색상 조합으로 소품 선택
- 스포트라이트: 반사광으로 색상 왜곡
  → 무광 소품 사용
- 그림자: 로봇 팔의 그림자가 물체 위에 떨어지면 인식 저하
  → 카메라/조명 각도로 해결
```

## 4. 로봇 세팅

### 작업 범위 확인

```
E0509 제원:
  Reach: 900mm
  Payload: 5kg

작업 범위 테스트:
  1. 약품 대기 구역의 각 약품 위치까지 도달 가능한지
  2. 조제함 각 슬롯까지 도달 가능한지
  3. Home 위치에서 모든 위치까지 경로에 충돌 없는지

  → 약품과 조제함을 로봇 reach 내에 배치
  → 권장: 베이스에서 40~70cm 거리
```

### Gripper 확인

```
현장에서 가장 먼저 확인:
  1. 그리퍼 종류? (Robotiq 2F / 두산 내장 / Pneumatic)
  2. 그리퍼 열림/닫힘 제어 방식? (I/O / ROS2 topic / DRCF)
  3. 그리퍼 최대 열림 폭? → 약품 크기에 맞는가?
  4. 그리퍼 힘 조절 가능? → 약병이 깨지지 않을 정도

  → configs/doosan_e0509_config.py의 GRIPPER_TYPE 업데이트
```

### Joint Limits 실측

```bash
# Teach pendant에서 각 joint를 끝까지 움직여보고 실측
# 결과를 config에 반영:

python -c "
from configs.doosan_e0509_config import *
print_config()
"
# 이 출력값과 실측값을 비교 → 차이 있으면 config 수정
```

## 5. 데모 수집 가이드

### 수집 전략

```
총 에피소드: 50~70개 (약 1.5~2시간)

물체별 배분:
  파란 약병 → 1번: 10개
  파란 약병 → 2번: 5개
  파란 약병 → 3번: 5개
  빨간 약병 → 2번: 10개
  빨간 약병 → 1번: 5개
  하얀 약병 → 3번: 10개
  노란 약상자 → 1번: 5개
  노란 약상자 → 2번: 5개
  ──────────────────────────
  합계: ~55개 + 여유 15개 (실패/불량 대비)
```

### 수집 절차

```
1. 약품을 대기 구역의 약간 다른 위치에 배치 (매번 ±5cm 변동)
2. 터미널에서 instruction 확인
3. [Enter] → hand-guiding으로 동작 수행:
   가만히 → 약품으로 이동 → 그리퍼 닫기 → 들기 → 슬롯으로 이동 → 놓기 → home
4. [Ctrl+C] → 저장
5. 약품 원래 위치로 리셋
6. 반복

수집 명령:
  python utils/doosan_recorder.py \
    --task "Pick up the blue medicine bottle and place it in dispensing slot 1" \
    --num-episodes 10
```

### 수집 품질 체크

```
좋은 데모:
  ✓ 일정한 속도 (급가속/급감속 없음)
  ✓ 직선에 가까운 경로 (불필요한 우회 없음)
  ✓ 물체 위에서 정확히 그리퍼 닫기
  ✓ 슬롯 위에서 정확히 그리퍼 열기
  ✓ 5~10초 내 완료

나쁜 데모 (제거):
  ✗ 10프레임 미만 (너무 짧음)
  ✗ 중간에 멈칫거림/흔들림
  ✗ 물체를 떨어뜨림
  ✗ 잘못된 슬롯에 넣음

데모 검증:
  python utils/doosan_recorder.py --verify --data-dir ./data/raw
```

## 6. 현장 세팅 순서 (타임라인)

```
[0:00 ~ 0:05] 테이블 위치 + 로봇 위치 확인
              → 로봇 reach 내에 작업 영역이 들어오는지

[0:05 ~ 0:10] 소품 배치
              → 약품 대기 구역 + 조제함 위치 결정
              → 마스킹 테이프로 위치 표시 (재현성)

[0:10 ~ 0:20] 카메라 설치
              → 삼각대 세팅 + 각도 조정
              → 화각 확인 (약품/조제함/그리퍼 전부 보이는지)
              → USB 케이블 정리 (로봇 동선 방해 없도록)

[0:20 ~ 0:25] 로봇 연결 + joint 테스트
              → ROS2 launch + joint_states 수신 확인
              → 약품 각 위치까지 수동 이동 → 도달 가능 확인
              → 그리퍼 동작 확인

[0:25 ~ 0:30] config 업데이트
              → CONTROLLER_IP, GRIPPER_TYPE, CAMERA_INDEX
              → joint limits 실측 반영

[0:30 ~     ] 데모 수집 시작
```

## 7. 트러블슈팅

### 물체를 못 잡음

```
원인 1: 그리퍼 열림 폭 < 물체 지름
  → 더 작은 물체로 교체

원인 2: 물체가 미끄러움
  → 고무 밴드 감기, 마스킹 테이프 붙이기

원인 3: VLA가 물체 위치를 잘못 인식
  → 카메라 각도 조정
  → 데모 추가 수집 (해당 물체 위치)
  → 조명 변경
```

### 색상 구분 안 됨

```
원인: 조명 환경에서 색상이 비슷하게 보임
  → 더 대비 높은 소품으로 교체
  → 물체에 라벨 부착 (글자 + 색상)
  → 카메라 화이트밸런스 수동 조정
```

### 조제함 위치를 헷갈림

```
원인: 슬롯 간 시각 차이 부족
  → 번호 라벨 크게 부착
  → 슬롯 간격 넓히기
  → 각 슬롯에 다른 색상 표시
```

### 로봇 동선에서 충돌

```
원인: 약품과 조제함 사이 경로에 장애물
  → 테이블 위 정리
  → 약품/조제함 위치 재배치
  → Home 위치 변경 (충돌 없는 자세)
```
