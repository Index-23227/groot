## 분석

### ICL이 효과적이었던 부분
- **FP 2건 해결** (base3, base8): ICL 예시로 "정렬됨처럼 보여도 실패" 패턴 학습
- **수치 기반 판단**: "X축 5% 이내", "Y차이 0.10 이내" 기준을 reason에 명시적으로 활용
- **Recall 100% 유지**: TP 케이스를 하나도 놓치지 않음

### ICL으로 해결하지 못한 부분
- **base9 FP 지속**: X=0%, Y차이≈0.10으로 기준값 경계 — Gemini가 가능으로 판단
  - 실제 실패 원인이 이미지만으로는 구별 불가능한 Z 깊이 문제
  - 해결책: 그리퍼 stroke 피드백을 실제 확인 루프에 추가

### base9 FP가 지속되는 이유
```
그리퍼 bbox: [0.45, 0.35, 0.15, 0.15]
캔 bbox:     [0.45, 0.45, 0.10, 0.20]
X 오프셋: 0% → 완벽한 정렬
Y 차이: 0.10 → 정확히 임계값
```
이미지상으로는 정렬된 것처럼 보이는 경계 케이스. 실제 실패 원인(Z 깊이 부족)은 2D 이미지에서 구별 불가.

    execute_pick()
    stroke = get_gripper_stroke()
