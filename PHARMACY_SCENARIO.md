# 약국 조제 보조 로봇 — 시나리오 상세

> **문제**: 한국 약사 1인당 하루 처방전 200~300건 처리.
> 조제 과정에서 약품 오인이 연간 수만 건 발생.
> 특히 야간/주말 1인 약국에서 피로 누적 시 위험.
>
> **해결**: 약사가 음성으로 지시하면 로봇이 약품을 시각으로 식별하여
> 정확한 조제함 슬롯에 배치. 약사는 상담/검수에 집중.

---

## 왜 VLA인가 — 기존 자동화와의 차이

### 기존 약국 자동화의 한계

```
현재 약국 자동화 = "자판기형 로봇" (BD Rowa, Omnicell 등)

  ┌──────────────────────────────────────┐
  │  대형 캐비닛 + 바코드 스캔 + 레일   │
  │                                      │
  │  비용: 1~3억 원                       │
  │  크기: 방 하나를 차지                  │
  │  약품 등록: 모든 약품을 바코드로       │
  │            사전 등록해야 함            │
  │  환경 변화: 약품이 바뀌면             │
  │            시스템 재설정 필요          │
  │  적용 범위: 대형 병원 약제실 전용      │
  │            동네 약국에는 도입 불가      │
  └──────────────────────────────────────┘

  → 한국 24,000개 약국 중 도입 가능한 곳: 극소수
  → 정작 도움이 필요한 1인 약국/야간 약국에는 적용 불가
```

### VLA가 해결하는 것

```
VLA 기반 접근 = "눈과 귀가 있는 조수"

  ┌──────────────────────────────────────────┐
  │  카메라 1대 + 협동 로봇 1대 + 마이크     │
  │                                          │
  │  1. 사전 등록 불필요                       │
  │     → VLA가 "파란 약병"을 보고 스스로 식별 │
  │     → 새 약품이 와도 language로 지시하면 됨 │
  │     → 바코드 인프라 불필요                 │
  │                                          │
  │  2. 자연어 인터페이스                      │
  │     → 약사: "파란 약병을 1번에 넣어줘"     │
  │     → 프로그래밍/설정 변경 zero            │
  │     → 약사의 학습 비용 zero                │
  │                                          │
  │  3. 위치 변동에 강건                       │
  │     → 약병이 정확한 위치에 없어도 인식     │
  │     → 실제 약국 환경은 항상 어수선함       │
  │     → 고정 좌표 기반 시스템에서는 불가능   │
  │                                          │
  │  4. 비용과 공간                            │
  │     → 카메라 + 로봇팔 = 기존 대비 1/10    │
  │     → 약국 카운터 위에 올려놓으면 끝       │
  │     → 1인 약국도 도입 가능                 │
  └──────────────────────────────────────────┘
```

### 왜 하드코딩 pick-and-place가 아닌가

```
"그냥 좌표 찍어서 옮기면 되지 않나?"

  하드코딩 방식:
    약병A → (x=0.3, y=0.1) → 슬롯1 (x=0.5, y=0.2)
    약병B → (x=0.3, y=0.2) → 슬롯2 (x=0.5, y=0.3)
    ...

  문제:
    ✗ 약품이 조금만 밀려도 못 잡음
    ✗ 새 약품 추가 시 매번 좌표 재설정
    ✗ 약사가 "3번에 넣어줘"라고 말해도 이해 불가
    ✗ 환경(조명, 배경) 변하면 실패

  VLA 방식:
    카메라로 보고 + 언어를 이해하고 + 행동을 생성
    → 위치 변동, 새 약품, 자연어 지시에 모두 대응
    → 이것이 "범용 조제 보조"가 가능한 이유
```

### 핵심 기술 차별점: Language Grounding

```
VLA의 진짜 강점은 language grounding:

  "파란 약병을 1번에" → 모델이 해야 할 것:
    1. "파란" → 영상에서 파란색 물체 식별 (vision)
    2. "약병" → 약병 형태 인식, 상자와 구분 (vision)
    3. "1번" → 조제함 1번 슬롯 위치 파악 (vision)
    4. 잡기 → 옮기기 → 놓기 동작 생성 (action)

  이 4가지를 하나의 모델이 end-to-end로 처리
  → 별도 object detector, planner, controller 불필요
  → 시스템이 단순하고, 새 상황에 유연하게 대응
```

## 왜 두산 E0509인가

### 약국 환경에 적합한 이유

```
약국 카운터 = 좁은 공간 + 사람 바로 옆 + 깨지기 쉬운 약품

  E0509 제원:
    DOF: 6축
    Reach: 900mm
    Payload: 5kg
    무게: ~18kg
    안전 등급: 협동 로봇 (ISO 10218, ISO/TS 15066)

  왜 이 로봇인가:

  1. 협동 로봇 (Cobot)
     → 안전 펜스 없이 약사 바로 옆에서 작동
     → 약사가 조제대에서 일하면서 로봇이 옆에서 보조
     → 산업용 로봇(안전 펜스 필수)은 약국에 불가

  2. 6축 = 약국 카운터에 딱 맞는 자유도
     → 약병 집기: 위에서 내려와서 잡기 (4축으로 가능)
     → 좁은 슬롯에 넣기: 각도 조절 필요 (6축 필요)
     → 장애물 회피: 약품 사이를 비켜가기 (6축 필요)
     → 7축은 과잉, 4축은 부족 → 6축이 최적

  3. Reach 900mm ≈ 약국 카운터 폭
     → 일반 조제 카운터: 60~100cm
     → 로봇 하나로 카운터 전체 커버 가능
     → 더 큰 로봇은 공간 낭비, 더 작으면 도달 못 함

  4. Payload 5kg >> 약품 무게
     → 약병: 50~200g, 약상자: 100~500g
     → 충분한 여유 → 안정적이고 정밀한 동작 가능

  5. 크기와 무게
     → 18kg → 카운터 위에 올려놓을 수 있음
     → 대형 산업 로봇 대비 설치 공사 불필요
     → 약국 리모델링 없이 도입 가능
```

### E0509 + VLA 시너지

```
  E0509 (하드웨어)          GR00T VLA (소프트웨어)
  ───────────────          ──────────────────────
  협동 로봇 → 약사 옆 OK    자연어 이해 → 약사 지시 OK
  6축 정밀 동작              end-to-end 행동 생성
  900mm reach               카메라 한 대로 전체 인식
  약국 카운터 설치           약품 사전 등록 불필요

              ┌─────────────────────┐
              │  결합하면:          │
              │  약국 카운터에 놓고  │
              │  카메라 하나 달면    │
              │  약사가 말로 시키는  │
              │  조제 보조 시스템    │
              │  완성               │
              └─────────────────────┘

  → "대형 인프라 없이, 동네 약국에도 도입할 수 있는
     저비용 AI 조제 보조" 라는 스토리
```

## 시뮬레이션

### Isaac Sim 시뮬레이션 (3D 물리 + 카메라 렌더링)

GPU가 있는 환경에서 Isaac Sim 기반 물리 시뮬레이션:

```bash
# 환경 테스트 (GUI 모드)
python sim/pharmacy_isaac_env.py

# Headless + Domain Randomization
python sim/pharmacy_isaac_env.py --headless --domain-rand

# 자동 데모 수집 (200개, DR 적용)
python sim/isaac_data_collector.py --num-episodes 200 --headless --domain-rand

# 특정 시나리오만
python sim/isaac_data_collector.py --scenario basic_single --num-episodes 50
```

### Isaac Sim → 학습 → 배포 파이프라인

```
Isaac Sim 데모 수집               학습                    배포
─────────────────    ──────────────────    ──────────────
sim/isaac_data_      utils/convert_to_    sim/sim2real_
collector.py         lerobot.py           deploy.py
     │                    │                    │
     ▼                    ▼                    ▼
./data/sim_raw/  →  ./data/lerobot/  →  실제 E0509 로봇
(200~500 에피소드)   (LeRobot v2 형식)   (Sim2Real 보정 적용)
```

```bash
# 1. Sim 데모 수집
python sim/isaac_data_collector.py --num-episodes 500 --headless --domain-rand

# 2. LeRobot v2 변환
python utils/convert_to_lerobot.py --data-dir ./data/sim_raw

# 3. GR00T 학습
bash scripts/04_train_groot.sh

# 4-a. Sim에서 먼저 평가
python sim/sim2real_deploy.py --mode sim-eval --scenario basic_single

# 4-b. Sim2Real 캘리브레이션 (실제 로봇 연결 시)
python sim/sim2real_deploy.py --mode calibrate

# 4-c. 실제 로봇 배포
python sim/sim2real_deploy.py --mode real \
  --instruction "Pick up the blue medicine bottle and place it in dispensing slot 1"
```

### 소프트웨어 시뮬레이션 (로봇 없이)

전체 파이프라인을 로봇 없이 검증:

```bash
# 기본 시나리오
python utils/pharmacy_sim.py --scenario basic_single

# 전체 시나리오 순회
python utils/pharmacy_sim.py --scenario all

# 실패 주입 테스트
python utils/pharmacy_sim.py --inject-failure stall    # stall → 재시도 → fallback
python utils/pharmacy_sim.py --inject-failure clamp    # over-clamp → 재시도 → fallback

# STT 시뮬레이션 (텍스트 입력)
python utils/pharmacy_sim.py --stt

# 파라미터 튜닝 테스트
python utils/pharmacy_sim.py --execute-horizon 6 --overlap 3 --decay 0.5
```

### 시뮬레이션이 검증하는 것

| 항목 | 검증 내용 |
|------|----------|
| MockVLA → TemporalBlender | chunk blending이 부드러운 동작을 만드는가 |
| ActionAdapter 3중 safety | delta/position/velocity clamp이 정상 동작하는가 |
| FailureDetector | stall/over-clamp을 정확히 감지하는가 |
| 재시도 로직 | blender reset 후 새 inference가 정상 동작하는가 |
| Fallback 전환 | 2회 실패 후 classical fallback으로 넘어가는가 |
| STT 파이프라인 | instruction 입력이 올바르게 전달되는가 |

## 레벨별 시연 포인트

### Level 0 — "동작합니다" (VLA가 실제 로봇을 움직인다)

```
하드코딩 instruction → E0509가 약병을 잡고 놓음

심사위원에게:
  "GR00T VLA 모델의 action 출력이 실제 6축 협동 로봇을 제어합니다.
   좌표 하드코딩이 아니라, 카메라 영상 + 언어를 입력받아
   end-to-end로 행동을 생성하는 것입니다."

기술 포인트: VLA → action chunk → safety clamp → E0509 servoj
```

### Level 1 — "정밀합니다" (약품을 다루는 섬세함)

```
TemporalBlender로 부드러운 동작 → 약병을 흔들리지 않게 옮김

심사위원에게:
  "약품은 액체가 들어있어 급가속하면 쏟아집니다.
   action chunk blending으로 관절 간 전환을 부드럽게 하여
   약품 취급에 적합한 정밀 동작을 구현했습니다.
   협동 로봇 E0509의 힘 제어와 결합하여 안전합니다."

기술 포인트: TemporalBlender + E0509 force limit
```

### Level 2 — "말로 지시합니다" (VLA의 L이 빛나는 순간)

```
약사가 마이크에 한국어로 말함 → Whisper → Claude → VLA

심사위원에게:
  "약사는 조제대에서 장갑을 끼고 있어 키보드를 칠 수 없습니다.
   음성으로 자연어 지시 → VLA가 언어를 이해하고 행동으로 연결.
   이것이 바코드 스캔이나 버튼 UI와 근본적으로 다른 점입니다.
   새 약품이 와도 이름만 말하면 됩니다."

기술 포인트: STT → LLM instruction 정제 → VLA language grounding
```

### Level 3 — "대화합니다" (약사-로봇 양방향 소통)

```
로봇이 TTS로 "파란 약병을 1번에 넣었습니다" 피드백

심사위원에게:
  "약사가 다른 일을 하면서도 로봇의 상태를 알 수 있습니다.
   실제 약국에서는 약사가 로봇을 계속 쳐다볼 수 없으므로
   음성 피드백이 필수적입니다."

기술 포인트: 동작 완료 감지 → TTS 피드백 루프
```

### Level 4 — "실수해도 복구합니다" (안전한 약국 보조)


## 사회적 맥락 — 발표 자료용

### 왜 의미 있는가

```
1. 약사의 단순 반복 노동 경감
   → 하루 200~300건 처방전 중 조제 배치는 반복 작업
   → 로봇에 위임하면 약사는 상담/검수에 집중
   → "약사를 대체"가 아니라 "약사의 전문성을 높인다"

2. 약품 오인 방지 — VLA만의 강점
   → VLA의 language grounding: "파란 약병"을 시각+언어로 식별
   → 사람: 피로 누적 → 200번째 처방에서 실수 가능
   → VLA: 1번째든 200번째든 동일한 정확도
   → 바코드 스캔 없이 시각만으로 → 인프라 투자 최소

3. 고령 사회 + 지방 약국 위기
   → 약국 수 감소 + 1인 약국 증가
   → 야간/주말 1인 운영 시 보조 인력 구할 수 없음
   → 기존 자동화(1~3억): 동네 약국에 도입 불가능
   → E0509 + 카메라: 기존 대비 1/10 비용, 카운터 위 설치

4. 확장성 — 같은 기술 스택으로
   → 약국 → 병원 조제실 (규모 확장)
   → 약국 → 물류 분류 (도메인 확장)
   → 언어 명령만 바꾸면 새 작업 → VLA의 범용성
```

### 수치 (발표 슬라이드용)

```
- 한국 약국 수: 약 24,000개 (2024)
- 약사 1인당 하루 처방전: 200~300건
- 약품 오인 사고: 연간 수만 건 (KOPS 보고)
- 1인 약국 비율: 증가 추세
- 야간 약국: 전체의 ~10%
- 기존 자동 조제 로봇 가격: 1~3억 원
- E0509 + 카메라 시스템: 기존 대비 1/10 수준
```

### 심사위원 예상 질문 대비

```
Q: "그냥 바코드 스캔하면 되지 않나?"
A: 바코드는 모든 약품을 사전 등록해야 하고, 스캐너 인프라가 필요합니다.
   VLA는 카메라 하나로 시각적으로 식별하므로 등록 없이 즉시 사용 가능합니다.
   또한 자연어로 지시할 수 있어 약사의 학습 비용이 제로입니다.

Q: "좌표 하드코딩하면 더 정확하지 않나?"
A: 고정 환경에서는 맞습니다. 하지만 실제 약국에서는:
   - 약품 위치가 매번 조금씩 다름 (약사가 내려놓는 위치 변동)
   - 새 약품이 수시로 추가됨
   - 하드코딩은 매번 재설정이 필요하지만, VLA는 그대로 동작합니다.

Q: "왜 E0509인가? 더 싼 로봇도 있지 않나?"
A: 약국은 약사 바로 옆에서 작동해야 하므로 협동 로봇(cobot)이 필수입니다.
   E0509는 ISO 10218 협동 로봇 안전 인증, 900mm reach로 카운터 전체 커버,
   6축으로 좁은 슬롯에도 정밀하게 넣을 수 있습니다.

Q: "시연은 단순한데, 실제 약국에서도 되나?"
A: 이번 시연은 concept proof입니다. 실제 적용에는:
   - 더 많은 약품 종류에 대한 학습 데이터
   - 약품 라벨 OCR 연동 (처방전 매칭)
   - 안전 인증 절차
   가 필요하지만, 핵심 기술(VLA + cobot)의 가능성을 보여주는 것이 목표입니다.
```
